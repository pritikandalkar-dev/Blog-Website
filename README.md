# Foodiee â€“ Recipe Blog Website

**Foodiee** is a user-friendly recipe blog website for food enthusiasts to discover & view recipes. Users can click to view full recipes, add their own comments using local storage. Built with HTML, CSS, JavaScript, and Bootstrap 5, it offers a responsive, modern interface with smooth navigation, Google Fonts (Quicksand & Poppins), and Font Awesome icons.


## Features

1. Recipe Cards: Display name, image, cooking time, servings, and date. 

2. View Recipe: Access detailed recipe instructions.

3. Comment Recipe: Users can leave comments on recipes.

4. Local Storage: Save favorite recipes in the browser.

5. Image-Based Blog Posts: Each category has visually appealing blog posts with images.

6. Recipe Filter Functionality: Filter recipes by All / Breakfast / Desserts / Mains using JavaScript.

7. Organized Folder Structure: Clean and structured folders for easier project maintenance.

8. Responsive Design: Mobile-friendly with Bootstrap 5 and flex layouts.

9. Navigation: Smooth scroll with active nav-link highlighting.

10. Modern UI: Integrated Google Fonts (Quicksand & Poppins) and Font Awesome icons for a professional look.


## Tech Stack

HTML, CSS, JavaScript, Bootstrap 5, Local Storage, Font Awesome

## Lessons Learned

#### What I Learned

While building Foodiee, I learned:

- How to dynamically render recipe data using JavaScript.

- Using local storage to save and retrieve user data like favorite recipes and comments.

- Implementing interactive features like viewing full recipes and adding comments.

- Building a responsive design with Bootstrap 5, flexbox layouts and Media Queries.

- Integrating Google Fonts and Font Awesome icons to enhance UI/UX.

- Adding scroll navigation with active link highlighting for better user experience.

#### Challenges and How I Overcame Them

1. Challenge: Storing and displaying user comments dynamically.
Solution: Learned how to structure objects in local storage and update the DOM when a new comment is added.

2. Challenge: Keeping the design responsive across devices.
Solution: Used Bootstrapâ€™s grid system, flex layouts, and media queries to ensure mobile-friendliness.

3. Challenge: Managing multiple recipe cards and view details functionality.
Solution: Used JavaScript event delegation and modular functions to handle click events efficiently.

4. Challenge: Persisting data without a backend.
Solution: Utilized local storage to save favorites and comments so data remains even after page reload.
